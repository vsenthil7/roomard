# Roomard Submission Media

Generated pitch deck (.pptx + .pdf) plus cover images, with **two independent
verifications** that prove the produced PDF says the right things on the right pages.

Mirrors the `roomard/demovideo/` pipeline (creation + verification-a + verification-b)
so the pattern is consistent across all submission artefacts. Ported from the
ATRIO Boardroom (AT-Hack0021) sibling project; re-themed to Roomard and re-written
with Roomard's real content.

## Folder layout

```
submission_media/
  README.md                                 # this file
  roomard-pitch-deck-{stamp}.pptx           # source (generated by scripts/build_pitch_deck.py)
  roomard-pitch-deck-{stamp}.pdf            # final deliverable (generated by scripts/pptx_to_pdf.ps1)
  cover-square-1200x1200.png                # social square
  cover-banner-1600x900.png                 # 16:9 hero banner
  cover-og-1200x630.png                     # Open Graph card
  _backup/                                  # backups, gitignored
  verification-a/                           # structural review (50 hard assertions, ~0.5s)
    structural-review.py
    run-verify-a.ps1
    reports/structural-review-{stamp}.txt + .json
  verification-b/                           # vision review (OCR + rubric, ~10s)
    extract-pages.py
    grade-pages.py
    grade_pages_lib.py
    run-verify-b.ps1
    requirements.txt
    reports/vision-review-{stamp}.json
```

## End-to-end pipeline

```powershell
# 1) BUILD the deck (run from roomard/ root)
python scripts/build_pitch_deck.py
#    -> submission_media/roomard-pitch-deck-{stamp}.pptx

# 2) CONVERT to PDF (PowerPoint COM, or LibreOffice fallback)
pwsh scripts/pptx_to_pdf.ps1
#    -> submission_media/roomard-pitch-deck-{stamp}.pdf

# 3) BUILD cover images
python scripts/build_cover_image.py
#    -> submission_media/cover-*.png

# 4) VERIFY-A - structural review (~0.5s, 50 hard assertions)
pwsh submission_media/verification-a/run-verify-a.ps1
#    -> verification-a/reports/structural-review-{stamp}.txt

# 5) VERIFY-B - vision review (~10s, OCR + rubric)
pwsh submission_media/verification-b/run-verify-b.ps1
#    -> verification-b/reports/vision-review-{stamp}.json
```

Both verifications passing on the same commit is the by-construction proof
that the deck is correct. We commit the verification reports alongside the PDF.

## Why this folder exists (not just trust the deck)

Same logic as `demovideo/`: an AI-generated artefact needs an automated
counter-signature before it goes in front of judges. The deck is built by
`build_pitch_deck.py` from project facts; a single content drift could silently
change a slide. verification-a catches "page missing / wrong page count / wrong
text". verification-b catches "right text in source, but renders blank or with
broken glyphs". Together they bound the risk.

## Re-using in future projects

```powershell
cp -r submission_media/verification-a /path/to/new-project/submission_media/
cp -r submission_media/verification-b /path/to/new-project/submission_media/
# Then edit two files:
#   verification-a/structural-review.py  -> swap EXPECTED_SLIDES
#   verification-b/grade_pages_lib.py     -> swap RUBRIC
```

Everything else (run-verify-*.ps1, extract-pages.py, grade-pages.py) is
project-agnostic.
